一.面对对象

面对对象是一种软件开发方法，是对现实世界的抽象和理解。面对对象的三大特性是封装、多态、继承。

1.封装

把数据和操作数据的方法绑定起来，通过接口对数据进行访问。

访问修饰符private，public，protected，default的区别：

public可允许当前类、同包、子类和其他包的数据访问，protected限制其他包的访问，默认情况下限制子类和其他包的访问，private只允许当前类对数据的访问。

2.多态

可理解为同样的对象调用了同样的方法做了不同的事情，多态分为运行时的多态的和编译时的多态。

override（重写）实现的是运行时的多态，它是对一个方法进行覆盖并重写，特点如下

​ （1）方法名、参数、返回值相同。

​ （2）子类不能缩小父类方法的访问权限。

​ （3）子类方法不能抛出比父类方法多的异常。

​ （4）只存在于父类和子类之间。

​ （5）被定义为final的方法不能进行重写。

overload（重载）是编译时的多态，是同一个类中的同名函数，但是参数个数和参数类型不同。

3.继承

从已有类信息创建新类的过程，在继承过程中不一定要使用父类的方法，可以在子类中定义新的方法。

4.面对对象六原则一法则

（1）单一职责原则：一个类只做它该做的事（高内聚）。

（2）开闭原则：软件实体应当对扩展开放，对修改关闭（当需要添加新的功能时只需派生新的类，不用修改原来

​ 的代码）。

（3）依赖倒转原则：面向接口编程。

（4）里式替换原则：任何时候都可以用子类型替换掉父类型（能使用父类型的地方一定可以使用子类型，可以用

​ 来检查继承关系是否合理）。

（5）接口隔离原则：接口要小而专不能大而全（高内聚的接口）。

（6）合成聚合复用原则：优先使用聚合或合成关系复用代码。

（7）迪米特法则：最少知识原则，一个对象应该对其他对象尽可能少的理解。
