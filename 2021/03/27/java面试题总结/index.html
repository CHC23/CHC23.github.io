java面试题总结

发表于 2021-03-27

1.jdk,jre,jvm区别和联系

jdk:java开发工具。包含jre和java工具（javac,java等）。

jre:java运行环境。包含bin(即jvm)和lib(java类库)

jvm：java虚拟机，把javac编译出的class文件根据java类库解释成机器码在不同平台运行，即跨平台的特性。

2.==和equals()的区别

==：比较的是堆栈中的值，基本类型比较的是栈中的变量值，引用类型比较的是堆中对象的地址。

equals：object中默认也是使用==实现的，不重写的话等价于==，一般都会重写，比如String中equals会重写（重写后比较的是对象的内容，即对象的值）。

3.final

1.final作用：
  • 修饰类：此类不可被继承。
  • 修饰方法：表示方法不可以被子类覆盖，但是可以被重载。
  • 修饰变量：表示变量一但被赋值就不可以重新改变它的值。

（1）修饰成员变量：

  • 修饰类变量：只能在静态初始化块中指定初始值或者在声明该类变量时指定初始值。
  • 修饰成员变量：可以在非静态初始化块中声明该变量或者在构造器中执行初始值。

（2）修饰局部变量：系统不会为局部变量进行初始化，局部变量必须由程序员显式初始化。因此在使用final修饰局部变量时，既可以在定义时指定默认值（后面不能再对其赋值），也可以不指定默认值而是在后面的使用中赋初值（仅一次）。

（3）修饰基本类型和引用类型：

  • 修饰基本类型：数值一旦初始化就不可以再次改变。
  • 引用类型：初始化后就不能再让其指向另一个对象。但是引用的值可以改变
2.为什么局部内部类和匿名内部类只能访问局部final变量

有内部类的文件编译之后会产生两个class文件，内部类和外部类处于同一级别，内部类不会因为定义在方法里面就会随方法的执行完而被销毁。

因此会产生一个问题：

当外部类的方法结束时，局部变量就会被销毁，但是内部类对象可能还存在（只有没有引用时才会死亡），这样就出现了内部类对象访问了一个已经死亡既不存在的变量这种情况。解决的方法是将局部变量复制了一份作为内部类的成员成员变量，当局部变量死亡之后，内部类仍然可以访问它，但是实际访问的该局部变量的“copy”值。

复制局部变量为内部类的成员变量时，必须保证这两个变量的值相同，即当我们改变内部类的成员变量时，方法中的局部变量也得跟着改变，解决的方法就是使用fianl修饰该局部变量，对其初始化之后就不允许修改其值，这样就保证了内部类的成员变量和方法的局部变量的一致性。事实上是一种妥协，使局部变量和内部类内建立的拷贝保持一致性。

4.String、StringBuffer、StringBuilder的区别和使用场景

String是用final修饰的，不可变，每次操作都会产生新的String对象。

StringBuffer和StringBuilder都是在原对象上进行操作。

StringBuffer的方法都是synchrinized修饰的，线程安全。StringBuilder线程不安全，在使用时需要额外使用synchrinized修饰保证线程安全。

性能：StringBuilder>StringBuffer>String。

使用场景：经常需要改变字符串内容时使用StringBuilder和StringBuffer，优先使用StringBuilder，当多线程使用共享变量时使用StringBuffer。

5.重载和重写的区别

重载：发生在同一个类中，是一个类的多态性，方法名必须相同，必须改变参数列表，参数类型不同、个数不同、顺序不同，返回值和访问修饰符可以不同，发生在编译时期

重写：发生在父子类之间，是父子类之间的多态性，方法名和参数列表必须相同，返回值范围小于等于父类，抛出异常的范围小于等于父类，访问修饰符大于等于父类；用private修饰父类的方法时，子类不可以重写该方法。（外壳不变，核心重写）

6.接口和抽象类

相同点

  • 都不可以被实例化。
  • 接口是对动作的抽象，抽象类是对根源的抽象（即事物的本质）。
  • 接口只有被其实现类实现了全部方法才可以被实例化，抽象类只有被其单继承的子类实现了其全部方法才能被实例化。

不同点：

  • 接口只能定义抽象方法（public abstract），不能实现方法。抽象类即可以定义抽象方法，也可以实现方法，也可以有构造方法。
  • 接口多实现，抽象类单继承。
  • 接口的成员变量只能是public static final类型的，抽象类的成员变量可以是各种类型。

7.List和Set的区别

List：有序，按照对象进入的顺序保存对象，可以重复对象，可以存放对个NULL对象。使用get（int index）获取指定下标的元素，还可以使用lterator取出所有元素载逐一遍历。

Set：无序存放，最多允许有一个Null对象，只能使用lterator取出所有元素再逐一遍历。
